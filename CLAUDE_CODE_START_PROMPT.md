# 项目启动提示词

在新的 Claude Code / AI 编程会话中打开本仓库后，粘贴以下内容作为开场：

```text
请先阅读仓库根目录的 CLAUDE.md 并作为项目上下文，特别是"实验设计红线"一节。

我在做一个思辨设计研究项目：AI Bureaucracy。

【项目是什么】
一个仿真的"AI 一体化在线政务服务平台"网页。办事大厅有 8 个窗口，
每个窗口由一个真实的 Claude API 调用值守——不是脚本，不是预写对话。
每个 Agent 只被赋予组织身份（科室、职责、边界、可开具的文书）、
组织条件（所有行为记入办件档案、对出具的文书负责、科室通讯录）、
以及少量与工作无关的人性化细节（工龄、爱好、近况）。

【研究问题】
官僚主义是人类文化的产物，还是组织结构的必然涌现？
我不预设答案——如果 Agent 根本不踢皮球，这本身就是重要发现。
所以绝对红线：任何 system prompt、工具描述里都不能出现引导官僚行为
的语句（不能写"要谨慎/要推诿/要多要材料/怕担责"）。改 lib/agents.ts
时要逐句自查。

【用户怎么互动】
1. 门户首页：现代政务门户视觉（庄重蓝+印章红），用户自由输入要办的事
   （不预设事项），生成办件编号，进入办事大厅。
2. 办事大厅：用户第一人称跑窗口——在某个窗口说明来意，Agent 可能
   回答、开具红头文书、列材料清单、让用户去别的窗口；用户只能亲自
   走到下一个窗口继续。Agent 之间还会互发内部函件（嵌套的真实模型
   调用），用户可以打开"内部流转记录"抽屉旁观。
3. 结束：任一 Agent 办结本件，或用户主动放弃。

【互动完之后】
报告页给两层反馈：
- 给办事人：办件回执（红章样式）；
- 给我（设计师/研究者）：办件统计（走过窗口数、被转办次数、内部
  函件数、被要求材料数…）、由组织之外的观察员调用生成的分析
  （列证据也列反证，引用档案原文，不预设"官僚主义一定出现"）、
  完整办件档案的 JSON/Markdown 导出。

【技术】
Next.js App Router + TypeScript + 纯 CSS；@anthropic-ai/sdk，
模型 claude-sonnet-5；ANTHROPIC_API_KEY 放 .env.local，绝不入库；
无数据库，办件状态在 localStorage。内部函件级联深度上限 3、
单轮调用上限 14（成本与死循环保护）。

【不要做的】
不要做成客服聊天机器人；不要预设固定办事流程；不要早期 Windows
或霓虹科幻皮肤；不要把 Agent 写成反派或蠢角色；荒诞感必须来自
"系统过于合理地运转"。
```
